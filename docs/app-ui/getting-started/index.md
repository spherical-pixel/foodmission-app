# App UI

> **Warning**: App UI is considered as an experimental product. It is provided **"as is"** without warranty of any kind, express or implied.

## Introduction

App UI is a powerful and flexible framework for building beautiful, high-performance user interfaces in Unity. The App UI framework is designed to help you create great apps with ease, by providing a set of UI components and patterns that you can use to quickly build and customize your app's interface.

With App UI, you can build apps for a wide range of platforms, including Android, iOS, Windows, MacOS, and the web. The framework is built on top of the Unity UI-Toolkit, which provides a powerful and flexible foundation for building UI elements that work seamlessly across all platforms.

Whether you're an experienced developer or just getting started with Unity, App UI has everything you need to create amazing user interfaces for your apps. From simple buttons and text boxes to more complex patterns like state management and navigation, the framework provides a comprehensive set of tools and best practices to help you build great UIs in no time.

## Overview of the App UI

The App UI package is a collection of UI components and patterns that can be used to build a user interface for your Unity project. But it also comes with a set of useful resources and tools.

### Components

Every UI component in the App UI package is defined as a [VisualElement](https://docs.unity3d.com/ScriptReference/UIElements.VisualElement.html) in [UI Toolkit](https://docs.unity3d.com/6000.0/Documentation/Manual/UIElements.html). You can find them in the `Unity.AppUI.UI` namespace.

### Features

The App UI package also provides a set of UI patterns that you can use to build more complex UIs.

- [Architecture](../references/architecture/mvvm-intro.md): Build your app using the MVVM architecture.
- [Observables](../references/architecture/observables.md): Bind your UI to data using observables.
- [Dependency Injection](../references/architecture/dependency-injection.md): Inject dependencies into your UI components and other objects.
- [Context Management](../references/contexts.md): Feed your UI with data from a global or scoped context.
- [Navigation](../references/navigation.md): Navigate between different screens in your app (useful for mobile apps).
- [Overlays](../references/overlays.md): Display UI elements on top of your app's content.
- [Localization](../references/localization.md): Localize your app's interface.
- [Accessibility](../references/accessibility.md): Make your app more inclusive.
- [Native Integration](../references/native-integration.md) (Android, iOS, MacOS, Windows)

### Resources

The App UI package comes with a set of useful resources that you can use to build your app's interface. These resources include:

- [Icons](../references/iconography.md)
- [Fonts](../references/typography.md)
- [Themes](../customization/theming.md)

### Tools and Samples

The package contains several tools and samples that you can use to build your app's interface.

- [UI Kit](../samples/ui-kit.md): A sample project that demonstrates how to use the App UI components and patterns.
- [Storybook](../samples/storybook.md): A tool that allows you to browse and test App UI or custom components and patterns.
- [Navigation](../samples/navigation.md): A sample project that demonstrates how to use the navigation pattern.
- [MVVM](../samples/mvvm.md): A sample project that demonstrates how to use the MVVM architecture.
- [Redux](../samples/redux.md): A sample project that demonstrates how to use the Redux architecture.
- [MVVM and Redux](../samples/mvvm-redux.md): A sample project that demonstrates how to use the MVVM and Redux architectures together.
- [Undo/Redo](../samples/undo-redo.md): A sample project that demonstrates how to use the undo/redo pattern.
